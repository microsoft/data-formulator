# 15.3 Agent 知识注入与搜索工具规划

> 状态：部分实现，规划中
> 最后更新：2026-05
> 前置文档：`15`（设计）、`15.1`（开发计划，已归档）、`15.2`（后续改进）
> 开发规范：`dev-guides/10-agent-knowledge-reasoning-log.md`

## 1. 核心结论

1. `rules`：`alwaysApply != false` 注入所有需遵守规则的 Agent system prompt；`alwaysApply: false` 只通过搜索注入。搜索自动跳过 always rules 避免重复。
2. `experiences`（原 skills + experiences 合并）：按任务搜索摘要注入，不全量注入。
3. `DataAgent` 已实现 `search_knowledge` / `read_knowledge` 工具，底层是 `KnowledgeStore.search()` 的分词子串匹配。
4. DuckDB 目前仅用于 catalog cache 表目录搜索，不用于 knowledge 搜索。
5. 知识搜索现阶段不需要 LLM 子 Agent，应先实现确定性后端搜索服务。

## 2. 已实现：注入边界修正

- `KnowledgeStore.load_always_apply_rules()` 加载 `alwaysApply != false` rules
- `DataAgent`、`DataRecAgent`、`DataTransformationAgent` 统一使用该 helper
- `DataAgent` 自动搜索排除已注入的 always rules（`search()` 中 `km.always_apply` → skip）
- `_tokenize_query()` 分词 + 停用词过滤 + `_match_score()` 多词加权匹配

## 4. 待实现：DuckDB 知识索引

### 4.1 目标

将 `KnowledgeStore.search()` 从 Python 遍历文件升级为 DuckDB 索引查询，提升搜索精度和可测试性。

实现能力：
1. 字段加权搜索（title/tags/description/body_prefix 分别计分）
2. 多词 query 匹配
3. category scope + always rules 过滤
4. find/grep/group 语义
5. 返回 `score`/`match_fields`/`matched_tokens` 便于调试
6. Python fallback 降级

索引字段：`category`、`path`、`title`、`description`、`tags_text`、`source`、`body_prefix`、`always_apply`、`created_at`、`updated_at`

### 4.2 `KnowledgeSearchService`

共享搜索服务，各 Agent 声明 categories 和 query source：

```python
class KnowledgeSearchService:
    def load_always_rules(self) -> list[KnowledgeHit]: ...
    def search_relevant(self, query, categories, *, exclude_always_rules=True, max_results=5) -> list[KnowledgeHit]: ...
    def format_summary_block(self, hits, heading) -> str: ...
```

各 Agent query source：
- `DataAgent`: user question + table names + thread summary
- `DataLoadingAgent`: latest user message + attachment context
- `InteractiveExploreAgent`: start question + table names
- `ChartInsightAgent`: chart type + field names + table names
- `ReportGenAgent`: report prompt + thread summaries

## 5. 待实现：`KnowledgeOrchestratorAgent`

任务编排器，决定搜索并行/串行、结果合并、预算控制：

| 职责 | 说明 |
|---|---|
| 任务拆分 | 按 categories 和 query source 拆子任务 |
| 并行调度 | 只读 find/grep/group 并发，全局并发上限 |
| 串行保护 | write/delete/distill/索引刷新串行 |
| 预算控制 | `max_results`/`max_snippet_chars`/`max_total_chars` + 截断标记 |
| 结果合并 | 去重 always rules、跨 category 排序、生成可注入摘要 |
| 降级 | DuckDB 不可用走 Python fallback |

## 6. 未来规划：上下文压缩与自动记忆

> 不属于本轮交付，但应预留接口。

| Agent | 触发条件 | 说明 |
|---|---|---|
| `ContextCompressionAgent` | 上下文达 80% window | 压缩会话上下文 |
| `KnowledgeBriefAgent` | 搜索结果超预算 | 压缩知识搜索结果为 brief |
| `MemoryCuratorAgent` | 会话完成/高价值分析链 | 判断是否更新 memory |
| `SkillDraftAgent` | 重复操作模式 | 提炼通用技能（pending 状态） |
| `RuleDraftAgent` | 稳定用户偏好 | 生成 rule 候选（必须用户确认） |

自动记忆写入策略：
- `experiences`：可自动写入（可配置）
- `skills`：半自动，建议确认
- `rules`：只生成候选，必须用户确认。`alwaysApply: true` 不可由模型静默创建
