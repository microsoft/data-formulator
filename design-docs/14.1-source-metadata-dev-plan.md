## 已完成并迁移到 dev-guides

### Phase 1：Schema 与通用流程

- `ColumnInfo` 已增加可选 `description` 字段，并保持旧 `workspace.yaml` 兼容。
- `TableMetadata.description` 作为源系统表级只读描述使用。
- `ExternalDataLoader.get_column_types()` 默认从 `get_metadata()` 保留表级 `description`。
- `ExternalDataLoader.ingest_to_workspace()` 已在 import 后 best-effort 合并表/列描述。
- `/api/tables/list-tables` 已返回 `table.description` 和 `columns[].description`。
- 测试覆盖在 `tests/backend/data/test_source_metadata.py`。

### Phase 2：高价值 Loader

- PostgreSQL：`obj_description` / `col_description`，轻量 catalog 不做逐表 sample/count。
- MySQL：`TABLE_COMMENT` / `COLUMN_COMMENT`，空 comment 归一化。
- BigQuery：使用 SDK 的 `table.description` / `field.description`。
- Superset：数据集和列级描述进入统一 `description` / `columns` 结构。

### Phase 3：其他 Loader

- MSSQL：`MS_Description` 扩展属性。
- Kusto：表和列 DocString。
- Athena/Glue：Glue table description 和 column comments。
- MongoDB、CosmosDB、S3、Azure Blob、Local Folder 没有稳定原生表/列描述，保持可选缺省。

### Phase 4：UX 展示

- Catalog 树对表/组描述展示 tooltip。
- Preview 标题附近展示表描述。
- Preview 筛选列选择器对列描述展示 tooltip。
- 已导入 Workspace 表区分只读 `systemDescription` 和用户可编辑 `attachedMetadata`。

### Phase 5：Agent 数据理解与发现

- `generate_data_summary()` 和 `get_field_summary()` 使用表/列源描述。
- `WorkspaceMetadata.search_tables()` 支持按表名、表描述、列名、列描述搜索。
- `catalog_cache/<source_id>.json` 支持用户级 catalog 搜索。
- `search_data_tables(query, scope)` 整合 workspace 和 catalog cache 两层结果。
- `inspect_source_data` 按表数量做渐进式上下文控制。
- 搜索结果不包含凭据、连接串、`loader_params` 或内部路径。
- 测试覆盖在 `tests/backend/data/test_phase5_agent_metadata.py`。

## 未完成或部分完成项

这些项仍需独立 PR 或明确删除需求：

1. `source_metadata_status`

   原计划要求 catalog metadata 暴露 `source_metadata_status` 状态指示器。代码中未实现该字段，也没有前端消费。

2. Catalog 树虚拟化

   原计划建议用 `react-arborist` 或 `@tanstack/react-virtual` 替换 MUI `SimpleTreeView`。当前仍使用 MUI tree view，虚拟化未落地。

3. Preview 表头列描述

   列描述已可在筛选列下拉 tooltip 中看到，但未在数据预览表头统一展示。

4. Workspace 主 UI 列级系统描述

   后端已返回 `columns[].description`，但前端 `DictTable.metadata` 尚未保存列级系统描述，主 UI 无通用展示入口。

5. DuckDB catalog cache 搜索

   当前实现是 Python JSON 子串搜索；DuckDB JSON 查询方案未落地，保留为未来优化而非当前需求。

6. 前端自动化测试

   后端和 Agent 测试已覆盖核心语义；未找到针对 Phase 4 UI 展示的专门前端测试。
