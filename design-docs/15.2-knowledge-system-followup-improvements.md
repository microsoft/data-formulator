## 仍需处理

### 1. 测试断言加强：`test_relevant_knowledge_injected`

搜索分词已实现，但测试仍使用弱断言：

```python
assert "[RELEVANT KNOWLEDGE]" in user_msg or agent._injected_knowledge == []
```

应改为强断言验证搜索确实命中了预期知识条目。

**依赖**：验证当前分词搜索是否能可靠命中测试数据中的 skill title "Handle Missing Values"（query "How to handle missing values?" 中的 `"missing"` / `"values"` 应命中）。如果能命中，直接加强断言；如果仍不能，需检查 `_match_score` 的权重阈值。

### 2. `list_all` / `search` 中文件读取风格统一

`list_all()` 和 `search()` 中 `md_file.read_text()` 直接使用 `Path.read_text()`，未通过 `jail.read_text(rel)`。安全影响为零（`rglob` 返回路径已在 jail 内），纯代码风格一致性问题。

改动：`md_file.read_text()` → `jail.read_text(rel_path)`，影响 `knowledge/store.py` 两处。

**优先级**：低。

### 3. 推理日志内容策略

`on` 模式当前记录 `user_question` 和 `query`，与原设计"on 模式不记录用户数据原文"不同。默认 `off` 时风险低。

- 方案 A（推荐）：接受现状，继续在 `dev-guides/10` 中明确 `on` 模式会记录问题文本
- 方案 B：收紧 `on` 模式，只记录摘要/hash/token 估算

取决于 `DF_AGENT_LOG=on` 是否会在多用户生产环境开启。
