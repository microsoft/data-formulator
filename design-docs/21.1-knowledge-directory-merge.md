## 概要

将原三目录 `rules` / `skills` / `experiences` 合并为两目录：

```
user_home/knowledge/
├── rules/            ← 手动创建，flat 文件，alwaysApply 语义
└── experiences/      ← 合并 skills + experiences，一层子目录 + 文件
```

关键改动：
- `VALID_CATEGORIES` 从三个缩减为 `{"rules", "experiences"}`
- `KnowledgeStore.search()` 自动跳过 `alwaysApply=true` 的 rules，避免与 system prompt 重复注入
- 注入文本使用语义标签 `[knowledge]` / `[rule]`，而非目录名
- `ExperienceDistillAgent` prompt 改为方法论导向（When to Use / Method / Pitfalls & Tips），并支持超长内容重试
- 前端从三 tab 简化为 Rules / Experiences 两个区块

实现细节已同步至 `dev-guides/10-agent-knowledge-reasoning-log.md`。

## 决策记录

| 编号 | 决策 | 理由 |
|------|------|------|
| D1 | 合并目录名用 `experiences` | 语义直观，与 ExperienceDistillAgent 命名一致 |
| D2 | 不引入 `kind` 分类字段 | 合并目录就是为了消除分类负担；再加 kind 分三种和之前三目录本质一样 |
| D3 | 用已有的 `source` 字段区分来源 | `manual` / `distill` / `auto` 足以区分手写和自动生成 |
| D4 | 不写迁移代码 | 尚未投产，无存量数据；减少复杂度 |
| D5 | rules 注入分两层，experiences 搜索统一不分类 | `alwaysApply` rules 通过 system prompt 注入（不走搜索）；非 always rules 仅 `DataAgent` 的搜索工具可按需命中；其余 Agent 只搜 experiences |
| D6 | rules 保持独立目录 | rules 有 alwaysApply 强语义、flat 目录、短 body 限制，与 experiences 差异大 |
| D7 | 搜索排除 alwaysApply rules | 避免与 `_load_knowledge_rules()` 的 system prompt 注入重复；搜索只返回非 alwaysApply 的 rules |
| D8 | 注入标签用 `[knowledge]` / `[rule]` 而非目录名 | LLM 不需要知道目录结构；语义标签帮助区分约束性规则和参考性知识 |
| D9 | ExperienceDistillAgent prompt 改为方法论导向 | 案例记录复用价值低；通用方法论（方法+踩坑规避）才是知识库的核心资产 |
| D10 | 超长内容自动重试一次 | LLM 输出超过 body 限制时追加精简指令重试，最多 1 次；降低用户看到报错的概率 |
| D11 | Agent 自建语言指令而非复用通用版 | 通用版字段列表是 DataAgent 专用的，经验提炼的输出结构不同，需要针对性的语言约束 |
